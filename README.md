# Transiting from Nginx Ingress to Envoy Gateway

## Introduction

The Kubernetes networking landscape is evolving. With Kubernetes SIG Network announcing the retirement of Ingress NGINX, which will receive best-effort maintenance only until March 2026, many teams are starting to evaluate the next generation of traffic management solutions.

In this session, Edison Wong will share how to transition from Nginx Ingress to Envoy Gateway and introduce the Gateway API, the modern replacement designed to provide more flexibility and extensibility for Kubernetes networking.

If you are operating Kubernetes clusters, planning platform modernization, or exploring the future of cloud-native networking, this session will provide practical insights into the migration path.
